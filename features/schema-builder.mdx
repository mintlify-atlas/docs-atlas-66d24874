Tusker's visual schema builder lets you create and modify PostgreSQL tables without writing DDL statements manually. Design your schema visually and see the generated SQL in real-time.

## Creating a new table

To create a table using the schema builder:

1. Press `Cmd/Ctrl+N` or click "New Table" in the sidebar
2. Select the target schema
3. Enter the table name
4. Add columns with the column builder
5. Define constraints and indexes
6. Preview the generated SQL
7. Execute with dry-run or apply directly

## Column builder

### Adding columns

Click the **Add Column** button to add a new column. Each column has:

- **Column name** - Identifier for the column
- **Data type** - PostgreSQL type from dropdown
- **Nullable** - Whether NULL values are allowed
- **Default value** - Optional default expression
- **Primary key** - Mark as primary key
- **Unique** - Add unique constraint
- **Foreign key** - Reference another table

### Custom types

Select **CUSTOM** from the type dropdown to specify:

- User-defined types
- Domain types
- Enum types (must be pre-created)
- Composite types
- Range types

Enter the exact type name as it exists in your database.

### Drag and drop reordering

Columns can be reordered by dragging:

1. Click and hold the grip icon (⋮⋮) on the left of any column row
2. Drag up or down to reorder
3. Release to drop in the new position

The generated SQL reflects the new column order.

<Info>
  Column order affects the physical storage layout in PostgreSQL and can impact query performance.
</Info>

## Constraints

### Primary key

Toggle the **Primary Key** checkbox on any column:

- Automatically sets `NOT NULL`
- Adds `PRIMARY KEY` constraint
- Only one column can be primary key (composite keys not yet supported)
- Disables nullable toggle

### Unique constraints

Toggle the **Unique** checkbox to add a unique constraint:

- Ensures no duplicate values in the column
- Creates a unique index automatically
- Can be combined with nullable columns

### Foreign keys

<Accordion title="Creating a foreign key">
  1. Check the **Foreign Key** checkbox
  2. Select the referenced schema
  3. Select the referenced table
  4. Select the referenced column (must be PRIMARY KEY or UNIQUE)
  5. Choose ON DELETE action: `NO ACTION`, `CASCADE`, `SET NULL`, `SET DEFAULT`, `RESTRICT`
  6. Choose ON UPDATE action (same options)
</Accordion>

The builder validates that the referenced column exists and is a valid foreign key target (PRIMARY KEY or UNIQUE).

### Check constraints

Check constraints are not yet supported in the visual builder. Use the SQL preview and edit mode, or create the table and alter it via the SQL playground.

## Indexes

### Creating an index

Click **Add Index** to create an index on your table:

1. Enter the index name (auto-generated by default)
2. Select the index method
3. Choose column or expression mode
4. Add columns to the index
5. Set sort direction and NULLS ordering

### Index methods

Tusker supports all PostgreSQL index types:

| Method | Use Case |
|--------|----------|
| **B-tree** | Default, general-purpose, supports sorting |
| **Hash** | Equality comparisons only, fast lookups |
| **GiST** | Geometric and full-text search |
| **GIN** | Full-text search, JSONB, arrays |
| **BRIN** | Very large tables with natural ordering |

### Index columns

For each index column, specify:

- **Column** - Select from table columns
- **Expression** - Or use a SQL expression (e.g., `LOWER(email)`)
- **Sort direction** - `ASC` or `DESC`
- **NULLS order** - `NULLS FIRST` or `NULLS LAST`

<Tip>
  Use expression indexes for case-insensitive searches or computed values.
</Tip>

### Multi-column indexes

Add multiple columns to a single index for composite indexes:

```sql
CREATE INDEX idx_users_last_first ON users(last_name, first_name);
```

Column order matters - the index is most effective when the leftmost columns are filtered.

## Live SQL preview

The bottom panel shows the generated SQL in real-time:

```sql
CREATE TABLE "public"."users" (
  "id" uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  "email" varchar(255) NOT NULL UNIQUE,
  "name" text NOT NULL,
  "created_at" timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX idx_users_email ON "public"."users"(LOWER(email));
```

The preview updates as you:
- Add or remove columns
- Change data types
- Modify constraints
- Add indexes

<Note>
  Click the copy icon to copy the SQL to your clipboard.
</Note>

## Execution modes

### Dry run

Click **Dry Run** to validate the SQL without applying changes:

- Checks for syntax errors
- Validates table and column names
- Ensures schema exists
- Reports potential conflicts
- No changes are made to the database

Dry run results show success or detailed error messages.

### Apply changes

Click **Apply** to execute the CREATE TABLE statement:

1. SQL is validated
2. Transaction begins
3. Table is created
4. Indexes are created
5. Transaction commits
6. New table tab opens automatically

If any step fails, the entire operation rolls back and no changes are made.

<Info>
  After successful creation, the schema builder tab closes and the new table opens for browsing and editing.
</Info>

## Modifying existing tables

The schema builder can also edit existing tables (coming in a future update). For now, use the SQL playground with `ALTER TABLE` statements:

```sql
ALTER TABLE users ADD COLUMN phone_number varchar(20);
ALTER TABLE users ALTER COLUMN email SET NOT NULL;
ALTER TABLE users ADD CONSTRAINT fk_company 
  FOREIGN KEY (company_id) REFERENCES companies(id);
```
